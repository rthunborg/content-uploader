Stena Content Portal — Design Directions

Six directions using only the Stena Line palette. Each shows the same two screens — the ambassador task list (phone) and the admin library (desktop) — so differences come purely from layout, chrome weight, and color application.

Typeface shown is a placeholder system font (brand typeface pending). Light Blue and Pink appear only because this is an internal tool. Green and Yellow are excluded per brand restrictions.

1 · Fleet Deck

warm · branded · calm

White base with beige panels exactly as the brand guide prescribes ("text box colour"). Blue carries every action; light blue marks selection and new items. The most literal translation of Stena's print identity into an app.

Choose this if: you want the friendliest, most unmistakably-Stena default with soft visual grouping.

2 · Gallery White

minimal · hairline · media-max

Chrome all but disappears: white everywhere, hairline borders instead of filled panels, color reserved exclusively for actions and states. The interface recedes so thumbnails and previews dominate every screen.

Choose this if: "media is the hero" should be taken to its logical extreme and the UI should feel like a gallery wall.

Ambassador · phone
Admin · desktop

3 · Dark Cockpit

pro tool · Lightroom energy

The whole app lives on brand Black with white text (a brand-approved 17.4:1 pairing). Media pops maximally; the brighter link-blue carries actions for contrast on dark. Reads as a professional media tool.

Choose this if: the admin experience is the soul of the product and ambassadors should feel like they're using "the studio's tool." Risk: heavier feel for a 90-second consumer moment.

Ambassador · phone
Admin · desktop

4 · Hybrid Stage

light app · dark where media is judged

Light, warm chrome (white + beige) for navigation, tasks, and management — but the media areas (library grid, triage queue, previews) sit on brand Black like a screening room. Matches the token architecture already defined in the foundation: surface-media only where content is evaluated.

Choose this if: you want Fleet Deck's warmth for people-facing screens and Dark Cockpit's media pop exactly where Petra judges content. (Facilitator's recommendation.)

Ambassador · phone (stays light)
Admin · desktop (dark stage)

5 · Card Deck

soft · rounded · consumer-warm

Inverts Fleet Deck: beige becomes the page itself, white cards float on it with big 16px radii and soft shadows. The friendliest, most consumer-app feel — consent cards and task cards look almost physical.

Choose this if: the ambassador experience should feel like a warm consumer app first and foremost. Risk: soft cards eat horizontal space in dense admin views.

Ambassador · phone
Admin · desktop

6 · Brand Bold

corporate · blue band · red accents

Strongest brand presence: a Core Blue app bar anchors every screen, selected states fill solid blue, and red appears as the attention accent (badges, counts) — closer to how blue and red dominate Stena's marketing.

Choose this if: stakeholders want the portal to feel like an official Stena flagship. Trade-off: using red for "new" weakens red's reserved destructive meaning, and heavy chrome competes with media.

How to evaluate

  • Layout intuitiveness — does the hierarchy put tasks (Jonas) and the triage entry (Petra) unmistakably first?
  • Interaction style — which chrome weight best supports one-motion-per-intent?
  • Visual weight — which density feels right for a 90-second phone moment vs. a 40-item desktop session?
  • Media focus — where do thumbnails pop most without the UI shouting?
  • Emotional goals — warm & safe for ambassadors, calm & fast for admins?
  • Brand alignment — enough Stena without becoming a marketing page?
